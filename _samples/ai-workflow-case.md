## Context

A 4-engineer team needed to triple their documentation output to support a product launch. Hiring another writer wasn't in budget.

## Problem

Engineers could write docs, but quality was inconsistent and each doc took 4–6 hours. The team needed a repeatable process that maintained quality at higher volume.

## Documentation process

1. **Audit** — Analyzed 20 existing docs for quality patterns and common issues
2. **Template design** — Created reusable templates for each doc type
3. **Prompt engineering** — Built prompt library for AI-assisted drafting
4. **Checklist** — Developed hallucination prevention checklist
5. **Rollout** — Trained 4 engineers, monitored first 10 docs through the process

## Deliverables

- Prompt library (12 templates covering research → draft → validation)
- Hallucination prevention checklist (8-point validation)
- Review process (peer review + technical review stages)
- Doc templates (API reference, guide, runbook, ADR)

## Result

Documentation output increased 3× while maintaining quality scores. Average time per doc dropped from 5 hours to 1.5 hours.
