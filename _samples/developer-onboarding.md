## Context

A 30-person engineering team had no structured onboarding. New hires pieced together setup instructions from a README, Slack messages, and asking around.

## Problem

Average time to first meaningful commit was 5 days. Every new hire hit the same setup issues, and existing engineers spent hours helping them debug.

## Documentation process

1. **Observation** — Shadowed 2 new hires through their first week
2. **Interviews** — Talked to 5 recent joiners about pain points
3. **Draft** — Created a step-by-step guide with checkpoints
4. **Testing** — Had the next new hire follow the guide on a fresh machine
5. **Iteration** — Fixed 12 issues found during live testing

## Deliverables

- 30-minute environment setup path (with Docker fallback)
- Repository tour (what lives where, naming conventions)
- First-PR checklist (from branch creation to merge)
- Team ownership map (who owns what, how to reach them)

## Result

Time to first commit dropped from 5 days to under 4 hours. Setup-related Slack questions dropped by 80%.
